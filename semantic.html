Заголовки и семантические теги

Semantic (семантика) related to meaning in language or logic (имеет какое то определенное значение в языке). Semantic HTML element is an element that implies (придаёт) some meaning to the content. То есть это элемент, который рассказывает что-то о контенте (важность, описание и тд)

  1. Заголовки

    Первый тип семантических тегов который мы рассмотрим это заголовки.
    <h1> … <h6> -- теги для заголовков. Важность заголовка определяется по убыванию, то есть <h1> -- самый важный заголовок, а <h6> -- наименее важный. Также по умолчанию у них разный размер шрифта
    Для стилизования эти размеры не используются. Все стили, размеры шрифта и ТД определяются CSSом. Заголовки разной степени важности важны для структуризации документа, не более. Почему просто не использовать <div>?? Так мы потеряем суть, не будет понятно,где заголовки. Во-вторых, для SEO важно, чтобы в h1 была самая нужная информация, слова описывающие контент.

  2. Тег header

    <header> содержит как правило информацию о странице, может быть логотип компании, список тегов (не HTML-тегов, а просто тегов, может быть навигацию и так далее.

  3. Тег nav

    Содержит контент, который используется для навигации внутри нашего веб-сайта.

  4. Тег section

    Тут тоже может содержаться какой-то контент, который разумно было бы разбить на секции. Например <article> https://ru.w3docs.com/uchebnik-html/html-teg-section.html

  5. Тег article

    https://ru.w3docs.com/uchebnik-html/html-teg-article.html

  6. Тег aside

    Содержит информацию, которая имеет непрямое отношение к странице, не такое прямое, как основной контент.

  7. Тег footer

    Нижний колонтитул.

Все вышеперечисленные теги являются блочными элементами. Вообще можно было бы тупо использовать везде div и не париться, НО если взглянуть на код , то куда проще читать код с осмысленными тегами.
